2. 문자열과 텍스트¶

  • start: 2014-10-02
  • end: 2014-10-03
  • 거의 모든 쓸만한 프로그램에는 어떻게든 데이터 파싱이나 결과물 출력과 같은 텍스트 프로세싱이 들어있음
  • 문자열 나누기, 검색, 빼기, 렉싱, 파싱과 같이 텍스트 처리와 관련있는 일반적인 문제에 초점
  • 대부분의 문제는 파이썬에 내장되어 있는 문자열 메소드로 해결 가능
  • 조금 복잡한 문제는 정규 표현식을 사용하거나 문자열 파서를 작성해야 됨
  • 조금 복잡한 유니코드도 다룸

2.1 여러 구분자로 문자열 나누기¶

문제¶

  • 문자열을 필드로 나누고 싶지만 구분자(그리고 그 주변의 공백)가 문자열에 일관적이지 않다.

해결¶

  • 문자열 객체의 split() 메소드는 아주 간단한 상황에서 사용하도록 설계
  • 여러개의 구분자나 구분자 주변의 공백까지 고려하지 않음
  • 좀 더 유연해져야 할 필요가 있다면 re.split() 메소드 사용
In [1]:
line = 'asdf fjdk; afed, fjek, asdf,      foo'
In [2]:
import re
In [3]:
re.split(r'[;,\s]\s*', line)
Out[3]:
['asdf', 'fjdk', 'afed', 'fjek', 'asdf', 'foo']

토론¶

  • re.split() 함수는 분리 구문마다 여러 패턴을 명시할 수 있다는 장점 존재
  • 앞의 코드에서 본 대로 쉼표(,), 세미콜론(;), 공백문자와 뒤이어 나오는 하나 이상의 공백문자 모두를 분리 구문으로 사용
  • 이 패턴이 나올 때마다 매칭된 부분은 모두가 구분자가 됨
  • 결과는 str.split()과 마찬가지로 필드 리스트가 됨
  • re.split()을 사용할 때는 괄호 안에 묶인 정규 표현식 패턴이 캡쳐 그룹(capture group)이 된다는 점에 주의
  • capture group을 사용하면, 매칭된 텍스트에도 결과가 포함됨
In [7]:
line
Out[7]:
'asdf fjdk; afed, fjek, asdf,      foo'
In [10]:
fields = re.split(r'(;|,|\s)\s*', line)
fields
Out[10]:
['asdf', ' ', 'fjdk', ';', 'afed', ',', 'fjek', ',', 'asdf', ',', 'foo']
  • 구분 문자만 추출해야 할 필요
  • 출력문을 재구성하기 위해서 필요한 경우
In [11]:
values = fields[::2]
values
Out[11]:
['asdf', 'fjdk', 'afed', 'fjek', 'asdf', 'foo']
In [13]:
fields[1::2]
Out[13]:
[' ', ';', ',', ',', ',']
In [12]:
delimiters = fields[1::2] + ['']
delimiters
Out[12]:
[' ', ';', ',', ',', ',', '']
In [14]:
# 동일한 구분자로 라인을 구성한다.
''.join(v+d for v, d in zip(values, delimiters))
Out[14]:
'asdf fjdk;afed,fjek,asdf,foo'
  • 구분자를 저장했다가 나중에 다시 사용한다. 단, 뒤에 덕지덕지 붙어있는 space 들은 모두 삭제!
  • 분리 구문을 결과에 포함시키지 않을 때: non-capture group(?:...)
In [15]:
# 구분자들은 저장되지 않음
re.split(r'(?:;|,|\s)\s*', line)
Out[15]:
['asdf', 'fjdk', 'afed', 'fjek', 'asdf', 'foo']
In [17]:
# () 캡쳐를 해서 최종 list에도 구분자가 포함 됨
re.split(r'(;|,|\s)\s*', line)
Out[17]:
['asdf', ' ', 'fjdk', ';', 'afed', ',', 'fjek', ',', 'asdf', ',', 'foo']

2.2 문자열 처음이나 마지막에 텍스트 매칭¶

문제¶

  • 문자열의 처음이나 마지막에 파일 확장자, URL scheme 등 특정 텍스트 패턴이 포함되었는지 검사하고 싶다

해결¶

  • 문자열의 처음이나 마지막에 패턴이 포함되었는지 확인하는 간단한 방법으로 str.startswith()나 str.endswith() 메소드가 존재

In [19]:

Out[19]:
True
In [20]:

Out[20]:
False
In [21]:
url = 'http://www.python.org'
In [22]:
url.startswith('http:')
Out[22]:
True
  • 여러 개의 선택지를 검사해야 한다면 검사하고 싶은 값을 튜플에 담아 startswith() 이나 endswith()에 전달

            return f.read()

2.3 쉘 와일드카드 패턴으로 문자열 매칭¶

문제¶

  • Unix 쉘에 사용하는 것과 동일한 와일드 카드 패턴을 텍스트 매칭에 사용하고 싶다(예: *.py, Dat[0-9]*.csv 등)

해결¶

  • fnmatch 모듈에 함수 fnmatch()와 fnmatchcase()가 존재
  • 난 정규표현식이 손에 익어서 그냥 regex로 하는게 좀 더 편한듯..
  • 그리고 와일드 카드보다 훨씬 많은 패턴을 적용할 수 있기 때문에 더 편리하다. 근데 속도 문제도 있고 하니 간단한 문제에서는 fnmatch도 나쁘지 않은 선택인듯
In [57]:
from fnmatch import fnmatch, fnmatchcase
In [58]:
fnmatch('foo.txt', '*.txt')
Out[58]:
True
In [59]:
fnmatch('foo.txt', '?oo.txt')
Out[59]:
True
In [60]:
fnmatch('Dat45.csv', 'Dat[0-9]*')
Out[60]:
True
In [61]:
names = ['Dat1.csv', 'Dat2.csv', 'config.ini', 'foo.py']
In [62]:
[name for name in names if fnmatch(name, 'Dat*.csv')]
Out[62]:
['Dat1.csv', 'Dat2.csv']
  • fnmatch()는 시스템의 파일 시스템과 동일한 대소문자 구문 규칙을 따름
In [63]:
# OS X(Mac)
fnmatch('foo.txt', '*.TXT')
Out[63]:
False
  • 이런 차이점이 마음에 들지 않는다면 fnmatchcase()를 사용하면 됨
  • 이 메소드는 지정한 소문자 혹은 대문자에 정확히 일치하는 것만 찾아냄
In [64]:
fnmatchcase('foo.txt', '*.TXT')
Out[64]:
False
  • 이 함수의 기능 중 간과하고 넘어가는 부분이 있는데, 바로 파일 이름이 아닌 데이터 프로세싱에도 사용할 수 있다는 사실
In [65]:
addresses = [
'5412 N CLARK ST',
'1060 W ADDISON ST',
'1039 W GRANVILLE AVE',
'2122 N CLARK ST',
'4802 N BROADWAY',
]
In [66]:
from fnmatch import fnmatchcase
In [67]:
[addr for addr in addresses if fnmatchcase(addr, '* ST')]
Out[67]:
['5412 N CLARK ST', '1060 W ADDISON ST', '2122 N CLARK ST']
In [70]:
[addr for addr in addresses if fnmatchcase(addr, '54[0-9][0-9] *CLARK*')]
Out[70]:
['5412 N CLARK ST']
In [72]:
[addr for addr in addresses if fnmatchcase(addr, '54[0-9][0-9]*CLARK*')]
Out[72]:
['5412 N CLARK ST']

토론¶

  • fnmatch가 수행하는 매칭은 간단한 문자열 메소드의 기능과 정규 표현식의 중간쯤 위치
  • 데이터 프로세싱을 할 때 간단한 와일드카드를 사용할 생각이라면 이 함수를 사용하는 것이 괜찮은 선택
  • 파일 이름을 찾는 코드를 실제로 작성해야 한다면 이 함수 대신 glob 모듈을 사용하도록 한다. 레시피 5.13 참고

2.4 텍스트 패턴 매칭과 검색¶

문제¶

  • 특정 패턴에 대한 텍스트 매칭이나 검색을 하고 싶다.

해결¶

  • 매칭하려는 텍스트가 간단하다면 str.find(), str.endswith(), str.startswith()와 같은 기본적인 문자열 메소드만으로도 충분
In [73]:
text = 'yeah, but no, but yeah, but no, but year'
In [74]:
# 정확한 매칭
text == 'yeah'
Out[74]:
False
In [75]:
# 처음이나 끝에 매칭
text.startswith('yeah')
Out[75]:
True
In [76]:
text.endswith('no')
Out[76]:
False
In [77]:
# 처음 나타난 곳 검색
text.find('no')
Out[77]:
10
  • 더 복잡한 매칭을 하려면 정규 표현식과 re 모듈을 사용
In [78]:
text1 = '11/27/2012'
text2 = 'Nov 27, 2012'
In [79]:
import re
In [80]:

else:
    print('no')

In [81]:
if re.match(r'\d+/\d+/\d+', text2):
    print('yes')

  • 동일한 패턴으로 매칭을 많이 수행할 예정이라면 정규 표현식을 미리 컴파일해서 패턴 객체로 만들어 놓는 것이 좋다

토론¶

  • 핵심이 되는 기능은 re.compile()을 사용해 패턴을 컴파일하고 그것을 match(), findall(), finditer() 등에 사용한다는 점
  • 패턴을 명시할 때 r'(\d+)/(\d+)/(\d+)' 와 같이 raw string을 그대로 사용하는 것이 일반적

  • match() 메소드는 문자열의 처음만 확인한다는 점에 주의

  • 예상되는 작업이 간단하지 않다면 패턴을 미리 컴파일해 두고 여러번 재사용하는 것이 더 효율적
  • 모듈 레벨 함수는 최근에 컴파일한 패턴을 기억하기 때문에 성능에 엄청나게 큰 차이를 보이지는 않음
  • 패턴을 미리 컴파일하면 추가적인 작업 등을 조금이라도 줄일 수 있음

2.5 텍스트 검색과 치환¶

문제¶

  • 문자열에서 텍스트 패턴을 검색하고 치환하고 싶다.

해결¶

In [110]:
text = 'yeah, but no, but yeah, but no, but yeah'
In [111]:
text.replace('yeah', 'yep')
Out[111]:
'yep, but no, but yep, but no, but yep'
  • 조금 더 복잡한 패턴은 re.sub()
In [112]:
text = 'Today is 11/27/2012. Pycon starts 3/13/2013.'
In [113]:
import re
In [114]:
# re.sub(pattern, repl, string, count=0, flags=0)
re.sub(r'(\d+)/(\d+)/(\d+)', r'\3-\1-\2', text)
Out[114]:
'Today is 2012-11-27. Pycon starts 2013-3-13.'
  • \3과 같은 표현은 패턴의 캡쳐 그룹을 참조
  • 동일한 패턴을 사용한 치환을 계속해야 한다면 컴파일링
In [115]:
import re
In [116]:
datepat = re.compile(r'(\d+)/(\d+)/(\d+)')
In [117]:
datepat.sub(r'\3-\1-\2', text)
Out[117]:
'Today is 2012-11-27. Pycon starts 2013-3-13.'
  • 더 복잡한 치환을 위해서 콜백 함수 명시
In [118]:
from calendar import month_abbr
In [120]:
list(month_abbr)
Out[120]:
['',
 'Jan',
 'Feb',
 'Mar',
 'Apr',
 'May',
 'Jun',
 'Jul',
 'Aug',
 'Sep',
 'Oct',
 'Nov',
 'Dec']
In [121]:
def change_date(m):
    mon_name = month_abbr[int(m.group(1))]
    return '{} {} {}'.format(m.group(2), mon_name, m.group(3))
In [122]:
# 이렇게 콜백 함수를 명시하는건 첨보네.. 신기하다.
datepat.sub(change_date, text)
Out[122]:
'Today is 27 Nov 2012. Pycon starts 13 Mar 2013.'

토론¶

  • sub() 메소드에 정규 표현식 검색과 치환 이외에 어려운 것은 없다.
  • 가장 이해하기 어려운 것이 정규 표현식 패턴을 만드는 것
  • 이건 독자들이 스스로 연습

2.7 가장 짧은 매칭을 위한 정규표현식¶

문제¶

  • 정규 표현식을 사용한 텍스트 매칭을 하고 싶지만 텍스트에서 가장 긴 부분을 찾아낸다. 만약 가장 짧은 부분을 찾아내고 싶다면 어떻게 해야 할까?

해결¶

  • 문장 구분자에 둘러싸여 있는 텍스트를 찾을 때 종종 발생(예를 들어 인용문 등)
In [171]:
str_pat = re.compile(r'\"(.*)\"')
In [172]:
text1 = 'Computer says "no."'
In [173]:
str_pat.findall(text1)
Out[173]:
['no.']
In [174]:
text2 = 'Computer says "no." Phone says "yes."'
In [175]:
str_pat.findall(text2)
Out[175]:
['no." Phone says "yes.']
  • 하지만 * 문자는 정규표현식에서 기본적으로 greedy 소비되므로 가장 긴 텍스트를 찾게 된다.
  • 이 문제를 해결하려면 *뒤에 ?를 붙여준다.
In [176]:
str_pat = re.compile(r'\"(.*?)\"')
In [177]:
str_pat.findall(text2)
Out[177]:
['no.', 'yes.']
In [196]:
# tag 찾을 때는 이렇게 해야 됨
# 중복되는 것들이 존재할 수도 있기 때문
str_pat = re.compile(r'(\")(.*?)\1')
str_pat.findall(text2)
Out[196]:
[('"', 'no.'), ('"', 'yes.')]

토론¶

  • .을 사용한 정규표현식을 작성할 때 가장 일반적으로 발생하는 문제를 보여줌
  • .은 newline을 제외한 모든 문자에 매칭
  • 하지만 점을 텍스트의 시작과 중간 사이에 넣으면, 매칭은 가장 긴 것을 찾아내려고 함
  • 결국 문장의 시작과 끝이 매칭에서 무시되고 더 긴 매칭에 하나로 포함되어 버림
  • 이떄 *나 +에 \?를 붙이면 정규표현식의 매칭 알고리즘에게 가장 짧은 것을 찾아내도록 명시할 수 있음

2.8 여러 줄에 걸친 정규 표현식 사용¶

문제¶

  • 여러 줄에 걸친 정규 표현식 매칭을 사용하고 싶다

해결¶

  • .을 사용한 텍스트 매칭을 할 때 이 문자가 개행문에 매칭하지 않는다는 사실을 잊었을 때 일반적으로 발생
In [197]:
comment = re.compile(r'/\*(.*?)\*/')
In [198]:
text1 = '/* this is a comment */'
In [199]:
text2 = '''/* this is a
              multiline comment */
'''
In [200]:
comment.findall(text1)
Out[200]:
[' this is a comment ']
In [201]:
comment.findall(text2)
Out[201]:
[]
  • [^*] 패턴도 사용할 수 있다.
In [202]:
comment = re.compile(r'/\*[^*]+\*/')
In [203]:
comment.findall(text2)
Out[203]:
['/* this is a\n              multiline comment */']
In [204]:
comment = re.compile(r'/\*((?:.|\n)*?)\*/')
In [205]:
comment.findall(text2)
Out[205]:
[' this is a\n              multiline comment ']
  • (?:.|\n)은 non-capture group: 매칭의 목적은 명시하지만 개별적으로 캡쳐하거나 숫자를 붙이지는 않음

토론¶

  • re.compile() 함수에 re.DOTALL 이라는 유용한 플래그를 사용
  • 이 플래그를 사용하면 .이 개행문을 포함한 모든 문자에 매칭(newline을 포함하기 위한 삽질을 안해도 되니 엄청 편리!)
In [206]:
comment = re.compile(r'/\*(.*?)\*/', re.DOTALL)
In [207]:
comment.findall(text2)
Out[207]:
[' this is a\n              multiline comment ']
  • re.DOTALL 플래그를 사용하면 간단한 패턴에는 잘 동작
  • 하지만 아주 복잡한 패턴을 사용하거나 여러 정규 표현식을 합쳐 토큰화를 한다거나 할 때 문제가 발생할 수 있음
  • 다른 선택의 여지가 있다면 플래그 없이 잘 동작할 수 있도록 정규 표현식을 작성하는 것이 더 좋음. 레시피 2.18 참고

문제¶

2.10 정규표현식에 유니코드 사용¶

문제¶

  • 텍스트 프로세싱에 정규 표현식을 사용중
  • 하지만 유니코드 문자 처리가 걱정됨

해결¶

  • 기본적인 유니코드 처리를 위한 대부분의 기능을 re모듈이 제공
  • \d는 유니코드 숫자에 이미 매칭함
In [239]:
import re
In [240]:
num = re.compile('\d+')
In [241]:
# ASCII number
num.match('123')
Out[241]:
<_sre.SRE_Match object; span=(0, 3), match='123'>
In [242]:
# arabia number
num.match('\u0661\u0662\u0663')
Out[242]:
<_sre.SRE_Match object; span=(0, 3), match='١٢٣'>
  • 특정 유니코드 문자를 패턴에 포함하고 싶으면, 유니코드 문자에 escape 시퀀스를 사용함
  • 아라비아 코드 페이지의 모든 페이지의 문자에 매칭하는 정규표현식은
In [243]:
arbic = re.compile('[\u0600-\u06ff\u0750-\u077f\u08a0-\u08ff]+')
  • 검색을 수행할 때, 텍스트를 노멀화하는 것이 좋은 접근법(레시피 2.9 참고)
  • 하지만 특별한 경우를 주의. 예를 들어, 대소문자를 무시하는 매칭에 대소문자 변환을 합친코드

  • 정말로 이 둘을 함께 사용해야겠다면, third-party regex 라이브러리를 설치하고 유니코드 대소문자 변환 등을 기본으로 제공하는 많은 기능을 이용해야 됨
