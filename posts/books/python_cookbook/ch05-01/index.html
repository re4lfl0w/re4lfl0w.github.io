5. 파일과 입출력¶

  • start: 2014-10-05
  • end: 2014-10-06
  • 모든 프로그램은 입력과 출력을 필요로 함
  • 여러 종류의 파일(텍스트, 바이너리, 파일 인코딩 등)을 다루는 일반적인 방법을 다룸
  • 또한 파일 이름과 디렉토리를 수정하는 방법도 배움

5.1 텍스트 데이터 읽고 쓰기¶

문제¶

  • 텍스트 데이터를 읽거나 써야 하는데 ASCII, UTF-8, UTF-16과 같이 서로 다른 인코딩을 사용해야 함

해결¶

  • 텍스트 파일을 읽기 위해 open() 함수에 rt 모드를 사용
In [1]:
# 파일 전체를 하나의 문자열로 읽음
# with open('somefile.txt', 'rt') as f:
#     data = f.read()
In [2]:
# 파일의 줄을 순환
# with open('somefile.txt', 'rt') as f:
#     for line in f:
        # 라인 처리
  • 마찬가지로 텍스트 파일을 쓰려면 wt 모드를 사용
  • 이 모드를 사용하면 모든 내용을 지우고 새로운 내용으로 덮어씀
In [3]:
# 텍스트 데이터 쓰기
# with open('somefile.txt', 'wt') as f:
#     f.write(text1)
#     f.write(text2)
In [4]:
# 리다이렉트한 print문
# with open('somefile.txt', 'wt') as f:
#     print(line1, file=f)
#     print(line2, file=f)
  • 파일의 끝에 내용을 추가하려면 at 모드로 open()을 사용함
  • 기본적으로 파일을 읽고 쓸 때 sys.getdefaultencoding()으로 확인할 수 있는 시스템 기본 인코딩을 사용함
  • 대부분의 컴퓨터는 이 기본 인코딩으로 utf-8을 사용
  • 읽고 쓸 텍스트가 다른 인코딩을 사용한다면 open()에 추가적인 encoding 인자를 전달
In [5]:
# with open('somefile.txt', 'rt', encoding='latin-1') as f:
  • 파이썬이 이해할 수 있는 텍스트 인코딩 종류는 수백 가지에 이름
  • 하지만 일반적으로 사용하는 인코딩은 ascii, latin-1, utf-8, utf-16이다.
  • 웹 애플리케이션을 만든다면 UTF-8이 안전한 형식
  • ascii는 U+0000에서 U+007F 범위의 7비트 문자에 일치
  • latin-1은 0-255바이트를 유니코드 U+0000에서 U+00FF로 매핑
  • latin-1 인코딩은 인코딩의 종류를 모르는 텍스트를 읽을 때 절대 에러가 발생하지 않는다는 점이 중요
  • 파일을 latin-1으로 읽으면 완벽하게 올바른 텍스트 디코딩을 만들어 내지 못할 수도 있다. 하지만 여전히 유용한 데이터를 추출할 것
  • 또한 데이터를 다시 돌려보낼 때 원본 입력을 보존함

토론¶

  • 테긋트 파일을 읽고 쓰는 과정은 그다지 어렵지 않음
  • 하지만 몇 가지 주의해야 할 점
  • 우선 예제에서 사용한 with 문이 파일을 사용할 콘택스트를 만든다.
  • 컨트롤이 with 블록을 떠나면 파일이 자동으로 닫힌다.
  • with 문을 꼭 사용하지 않아도 되지만, 그럴때는 반드시 파일을 닫아야 함
In [7]:
f = open('05/getting_a_directory_listing/example.py', 'rt')
data = f.read()
f.close
Out[7]:
<function TextIOWrapper.close>
  • 그리고 Unix와 Windows에서 서로 다른 줄바꿈 문자에 주의(\n과 \r\n)
  • 기본적으로 파이썬은 universal newline 모드로 동작
  • 이 모드는 일반적인 모든 줄바꿈을 알아보고, 읽을 때 모든 줄바꿈 문자를 \n으로 변환
  • 출력시에는 줄바꿈 문자를 \n을 시스템 기본 문자로 변환한다.
  • 이런 자동 변환을 원하지 않는다면 newline='' 인자를 open()에 넣어준다.
In [8]:
# 줄바꿈 문자 없이 읽기
# with open('somefile.txt', 'rt', newline='') as f:
  • 차이점을 보기 위해 Unix 컴퓨터에서 Windows 형식으로 인코딩된 텍스트 파일을 읽어 보겠다.

5.2 파일에 출력¶

문제¶

  • print() 함수의 결과를 파일에 출력

해결¶

  • print()에 file 키워드 인자 사용
In [66]:
!cat 05/hello_windows.txt

Hello World!
Hello World2!
In [65]:
with open('05/hello_windows.txt', 'wt') as f:
    print('Hello World!', file=f)
    # f.write랑 무슨 차이냐..?
    f.write('Hello World2!')

토론¶

  • 파일에 출력하기에서 이 이상의 내용은 없음
  • 하지만 파일을 텍스트 모드로 열었는지 꼭 확인해야 함
  • 바이너리 모드로 파일을 열면 출력에 실패함

5.3 구별자나 종단 부호 바꾸기¶

문제¶

  • print()를 사용해 데이터를 출력할 때 구분자나 종단 부호(line ending)를 바꾸고 싶다.

  • print()로 출력 시 아이템을 구분하는 문자를 스페이스 공백문 이외로 바꾸는 가장 쉬운 방법은 구별자를 지정하는 것
  • 어떤 프로그래머는 동일한 목적으로 str.join()을 사용하기도 함

5.4 바이너리 데이터 읽고 쓰기¶

문제¶

  • 이미지나 사운드 파일 등 바이너리 데이터를 읽고 써야 함

해결¶

  • open 함수에 rb와 wb모드를 사용해 바이너리 데이터를 읽거나 쓴다.
In [89]:

    f.write(b'Hello World')

  • 마찬가지로 데이터를 쓸 때도 바이트로 표현할 수 있는 형식의 객체를 제공해야 한다(바이트 문자열, bytearray 객체 등)

  • 바이너리 데이터를 읽을 때, 바이너리 문자열과 텍스트 문자열 사이에 미묘한 문법 차이가 있다.

5.5 존재하지 않는 파일에 쓰기¶

문제¶

  • 파일이 파일 시스템에 존재하지 않을 때, 데이터를 파일에 쓰고 싶다.

해결¶

  • 이 문제는 open()에 x모드를 사용해서 해결
  • w 모드와 다르게 x 모드는 잘 알려지지 않음
In [116]:
with open('05/xmode.txt', 'wt') as f:
    f.write('Hello\n')

with open('05/xmode.txt', 'xt') as f:
    f.write('Hello\n')

  • 이 레시피는 파일을 쓸 때 발생할 수 있는 문제점(실수로 파일을 덮어쓰는 등)을 아주 우아하게 피해가는 법을 알려줌

  • x 모드는 파이썬3의 확장 기능임을 기억
  • 이전 버전의 파이썬이나 파이썬 구현에서 사용하는 C 라이브러리는 이 모드를 지원하지 않음

5.6 문자열에 입출력 작업하기¶

문제¶

  • 파일같은 객체에 동작하도록 작성한 코드에 텍스트나 바이너리 문자열을 제공하고 싶다.

해결¶

  • io.StringIO()와 io.BytesIO() 클래스로 문자열 데이터에 동작하는 파일 같은 객체를 생성
In [122]:
import io
In [123]:
s = io.StringIO()
In [124]:
s.write('Hello World\n')
Out[124]:
12
In [127]:
print('This is a test', file=s)
In [128]:
# 기록한 모든 데이터 얻기
s.getvalue()
Out[128]:
'Hello World\nThis is a test\n'
In [129]:
# 기존 문자열을 파일 인터페이스로 감싸기
s = io.StringIO('Hello\nWorld\n')
In [130]:
s.read(4)
Out[130]:
'Hell'
In [131]:
s.read()
Out[131]:
'o\nWorld\n'
  • io.StringIO 클새느느 텍스트에만 사용해야 함
  • 바이너리 데이터를 다룰 때는 io.BytesIO 클래스 사용
In [132]:
s = io.BytesIO()
In [133]:
s.write(b'binary data')
Out[133]:
11
In [134]:
s.getvalue()
Out[134]:
b'binary data'

토론¶

  • 일반 파일 기능을 흉내 내려 할 때 StringIO와 BytesIO 클래스가 가장 유용
  • 예를 들어 유닛 테스트를 할 때, StringIO로 테스트 데이터를 담고 있는 객체를 만들어 일반 파일에 동작하는 함수에 사용
  • StringIO와 BytesIO 인스턴스가 올바른 정수 file-descriptor를 가지고 있지 않다는 점을 기억
  • 따라서 file, pipe, socket 등 실제 시스템 레벨 파일을 요구하는 코드에는 사용할 수 없음

import os
import mmap

def memory_map(filename, access=mmap.ACCESS_WRITE):
    size = os.path.getsize(filename)
    fd = os.open(filename, os.O_RDWR)
    return mmap.mmap(fd, size, access=access)

  • m[0]은 원래 encoding이고 v는 unsigned integer 다.
  • 그래서 unsigned integer 형태로도 보여줄 수 있다라는걸 말하고 싶은듯
  • 파일을 메모리 매핑해도 파일 전체가 메모리에 읽어 들여지지 않느다는 점을 기억
  • 이말인즉슨,메모리 버퍼나 배열에 복사하지 않는다는 의미
  • 운영체제는 그 대신 파일 내용을 위한 가상 메모리 섹션을 준비해 놓음
  • 서로 다른 지역에 접근하면 파일의 그 부분을 읽고 메모리에 매핑한다.
  • 하지만 절대 접근하지 않는 파일의 다른 부분은 그대로 디스크에 남아 있음. 이런 과정은 모두 자동
  • 하나 이상의 파이썬 인터프리터가 동일한 파일을 매핑하면, mmap 객체로 인터프리터 간 데이터를 교환 가능
  • 따라서 모든 인터프리터가 데이터를 동시에 읽고 쓸 수 있고, 한 쪽 인터프리터에서 데이터를 수정하면 다른 쪽에 자동으로 그 내용이 반영됨.
  • 물론 동기화를 위해 신경써야 할 부분이 없지는 않지만, 이런 접근법은 파이프나 소켓에서 메시지를 주고받는 대안으로 사용됨
  • Unix와 Windows에서 모두 동작하도록 일반적인 목적으로 작성됨
  • 하지만 mmap() 호출할 때 플랫폼마다 약간 다른 점이 있음에 주의
  • 추가적으로, 익명으로 매핑된 메모리 지역을 만드는 옵션이 있다. 이게 궁금하면 찾아봐!
