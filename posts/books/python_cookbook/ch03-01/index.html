3. 숫자, 날짜, 시간¶

  • start: 2014-10-03
  • end: 2014-10-04
  • 파이썬에서 정수와 부동 소수점을 써서 수학적으로 계산하기는 어렵지 않음
  • 하지만 분수, 배열, 날짜, 시간 계산은 조금 복잡

3.1 반올림¶

문제¶

  • 부동 소수점 값을 10진수로 반올림하고 싶다.

해결¶

  • 간단한 반올림은 내장 함수인 round(value, ndigits) 함수 사용
In [1]:
round(1.23, 1)
Out[1]:
1.2
In [2]:
round(1.27, 1)
Out[2]:
1.3
In [3]:
round(-1.27, 1)
Out[3]:
-1.3
In [4]:
round(1.25361, 3)
Out[4]:
1.254
  • 값이 정확히 두 선택지의 가운데 있으면 더 가까운 짝수가 됨
  • 예를 들어 1.5와 2.5는 모두 2가 됨
  • round(0에 전달하는 자릿수는 음수가 될 수 있음
  • 이 경우에는 10의 자리, 100의 자리 등의 순으로 자릿수가 결정됨
In [5]:
a = 1627731
In [6]:
round(a, -1)
Out[6]:
1627730
In [9]:
round(a, -2)
Out[9]:
1627700
In [10]:
round(a, -3)
Out[10]:
1628000

토론¶

  • 반올림과 서식화를 헷가릴지 않도록 주의하자
  • 특정 자릿수까지 숫자를 표현하는 것이 목적이라면 round()를 사용하는 것이 아니라 서식화를 명시하기만 하면 됨
In [11]:
x = 1.23456
In [12]:
format(x, '0.2f')
Out[12]:
'1.23'
In [13]:
format(x, '0.3f')
Out[13]:
'1.235'
In [14]:
'value is {:0.3f}'.format(x)
Out[14]:
'value is 1.235'
  • 또한 정확도 문제를 "수정하려고" 부동 소수점을 반올림하는 방법도 지양
In [16]:
a = 2.1
In [17]:
b = 4.2
In [18]:
c = a + b
In [19]:
c
Out[19]:
6.300000000000001
In [21]:
c = round(c, 2)
c
Out[21]:
6.3
  • 부동 소수점 계산을 하는 대부분의 애플리케이션에서 이와 같은 코드는 불필요
  • 계산에서 작은 오류가 발생했지만 이 정도 오류는 용인할 만한 것
  • 하지만 이런 오류를 절대적으로 피해야 한다면(금융 애플리케이션 등) decimal 모듈을 사용하는 것이 좋다.

3.2 정확한 10진수 계산¶

문제¶

  • 정확한 10진수 계산을 해야 하고, 부동 소수점을 사용할 때 발생하는 작은 오류를 피하고 싶다

해결¶

  • 부동 소수점 값에는 10진수를 아주 정확히 표현하지 못한다는 문제 발생
In [22]:
a = 4.2
In [23]:
b = 2.1
In [24]:
a + b
Out[24]:
6.300000000000001
In [25]:
(a + b) == 6.3
Out[25]:
False
  • 이런 오류는 CPU와 IEEE 754로 부동 소수점 숫자 계산을 할 때 필연적으로 발생
  • 파이썬의 부동 소수점 값이 바로 이 표현식을 사용하기 때문에 float를 사용해서는 이 오류를 피할 수 있는 방법이 없다.
  • 하지만 좀 더 정확한 계산을 하고 싶다면(성능 측면을 희생할 용의가 있다면), decimal 모듈을 사용해야 함
In [26]:
from decimal import Decimal
In [27]:
a = Decimal('4.2')
In [28]:
b = Decimal('2.1')
In [29]:
a + b
Out[29]:
Decimal('6.3')
In [30]:
print(a + b)

6.3
In [31]:
(a + b) == Decimal('6.3')
Out[31]:
True
In [32]:
(a + b) == 6.3
Out[32]:
False
  • 숫자를 문자열로 표현하는 이 모듈이 이상하게 보일지도 모르겠다.
  • 하지만 Decimal 객체는 우리가 기대하는 모든 동작(숫자 계산 포함)을 정확히 수행
  • 문자열 서식화 함수에 사용하거나 출력하면 마치 일반적인 숫자인 것처럼 보인다.
  • decimal의 중요한 기능으로 반올림의 자릿수와 같은 계산적 측면을 조절할 수 있다는 점
  • 여자만 변신하냐? 파이썬도 변신해 보자!: context가 뭔지 이해가 안되서 검색했더니 내가 번역한 문서가 나오네.. 그때 작성했을 때는 다 이해했다고 생각했지만 그게 아니었다!
In [33]:
# context: 환경설정
from decimal import localcontext
In [34]:
a = Decimal('1.3')
In [35]:
b = Decimal('1.7')
In [36]:
print(a / b)

0.7647058823529411764705882353
In [37]:
with localcontext() as ctx:
    # precision: 정확도
    ctx.prec = 3
    print(a / b)

0.765
In [38]:
help(localcontext)

Help on built-in function localcontext in module decimal:

localcontext(...)
    localcontext(ctx=None) - Return a context manager that will set the default
    context to a copy of ctx on entry to the with-statement and restore the
    previous default context when exiting the with-statement. If no context is
    specified, a copy of the current default context is used.
In [42]:
with localcontext() as ctx:
    ctx.prec = 50
    print(a / b)

0.76470588235294117647058823529411764705882352941176

토론¶

  • decimal 모듈은 IBM의 "General Decimal Arithmetic Specification"을 구현. 물론 이 책에서 다 다룰 수 없는 방대한 양의 옵션이 존재
  • float가 제공하는 17자리 정확도로 표현 못할 것은 없을것. decimal은 속도 느리니 되도록 쓰지 말것
  • 오류를 완전히 무시할 수는 없다.
  • 아주 크고 작은 숫자를 더할 때 발생하는 오류도 조심
In [43]:
nums = [1.23e+18, 1, -1.23e+18]
In [45]:
# 1이 사라진다.
sum(nums)
Out[45]:
0.0
In [46]:
nums2 = [1.23e+18, 1]
sum(nums2)
Out[46]:
1.23e+18
In [47]:
# math.fsum()을 사용하면 더 정확한 계산
import math
print(math.fsum(nums))
print(math.fsum(nums2))

1.0
1.23e+18
  • 하지만 또 다른 알고리즘에서 어떤 오류가 발생할지 알 수 없는 문제이므로 미리미리 공부해야 함
  • decimal 모듈을 사용하는 가장 큰 이유: 여러 문제를 피하기 위해서
    • 금융 등 숫자 계산을 철저히 하는 프로그램에서 오류가 발생하면 치명적
  • 따라서 금융 애플리케이션에서는 반드시 decimal 모듈 사용
  • 또한 데이터베이스를 다루는 파이썬 인터페이스도 Decimal 객체를 빈번하게 사용

3.3 출력을 위한 숫자 서식화¶

문제¶

  • 출력을 위해 자릿수, 정렬, 천 단위 구분 등 숫자를 서식화하고 싶다.

해결¶

  • 출력을 위해 숫자를 서식화하려면 내장 함수인 format()을 사용
In [48]:
x = 1234.56789
In [49]:
# 소수점 둘째 자리 정확도
format(x, '0.2f')
Out[49]:
'1234.57'
In [50]:
# 소수점 한 자리 정확도로 문자 10개 기준 오른쪽에서 정렬
format(x, '>10.1f')
Out[50]:
'    1234.6'
In [51]:
# 왼쪽에서 정렬
format(x, '<10.1f')
Out[51]:
'1234.6    '
In [52]:
# 가운데 정렬
format(x, '^10.1f')
Out[52]:
'  1234.6  '
In [53]:
# 천 단위 구분자 넣기
format(x, ',')
Out[53]:
'1,234.56789'
In [54]:
format(x, '0,.1f')
Out[54]:
'1,234.6'
  • 지수 표현법을 사용하려면 f나 e나 E로 바꾸면 됨
In [55]:
format(x, 'e')
Out[55]:
'1.234568e+03'
In [56]:
format(x, '0.2E')
Out[56]:
'1.23E+03'
  • 너비와 자릿수를 나타내는 일반적인 형식은 '[<>^]?너비[,]?(.자릿수)?'이다.
  • 이때 너비와 자릿수는 정수형으로 표시하고 ?는 선택사항임을 의미
  • .format() 메소드에도 동일한 서식을 사용
In [57]:
'The value is {:0,.2f}'.format(x)
Out[57]:
'The value is 1,234.57'
In [58]:
'The value is {:0.2f}'.format(x)
Out[58]:
'The value is 1234.57'

토론¶

  • 출력을 위한 숫자 서식화는 댁 간단
  • 앞에 소개한 기술은 부동 소수점 값과 decimal 모듈의 숫자에 모두 잘 동작
  • 자릿수를 제한하면 round() 함수와 동일한 규칙으로 반올림
In [59]:
x
Out[59]:
1234.56789
In [60]:
format(x, '0.1f')
Out[60]:
'1234.6'
In [61]:
format(-x, '0.1f')
Out[61]:
'-1234.6'
  • 천 단위 구분자는 지역 표기법을 따르지 않는다.
  • 이를 염두에 둔다면 locale 모듈의 함수를 사용해야 함
  • 문자열의 translate() 메소드를 사용하면 구분자 문자를 변경할 수도 있음
In [64]:
swap_separators = { ord('.'):',', 
                    ord(','):'.' }
In [65]:
format(x, ',').translate(swap_separators)
Out[65]:
'1.234,56789'
In [66]:
'%0.2f' % x
Out[66]:
'1234.57'
In [67]:
'%10.1f' % x
Out[67]:
'    1234.6'
In [68]:
'%-10.1f' % x
Out[68]:
'1234.6    '
  • 이 방식을 사용해도 괜찮지만 format()을 사용하는 것만큼 기능이 많지는 않음
  • 예를 들어 천 단위 구분자를 넣는 등의 기능은 %연산자가 지원하지 않음

3.4 2진수, 8진수, 16진수 작업¶

문제¶

  • 숫자를 2진수, 8진수, 16진수로 출력해야 한다.

해결¶

  • 정수 -> 2진수: bin()
  • 8진수 -> oct()
  • 16진수 -> hex()
In [69]:
x = 1234
In [70]:
bin(x)
Out[70]:
'0b10011010010'
In [71]:
oct(x)
Out[71]:
'0o2322'
In [72]:
hex(x)
Out[72]:
'0x4d2'
  • 앞에 0b, 0o, 0x가 붙는 것이 싫으면 format() 함수를 사용해도 됨
In [73]:
format(x, 'b')
Out[73]:
'10011010010'
In [74]:
format(x, 'o')
Out[74]:
'2322'
In [75]:
format(x, 'x')
Out[75]:
'4d2'
  • 정수형은 부호가 있는 숫자이므로, 음수를 사용하면 결과물에도 부호가 붙는다.
In [76]:
x = -1234
In [77]:
format(x, 'b')
Out[77]:
'-10011010010'
In [78]:
format(x, 'x')
Out[78]:
'-4d2'
  • 부호가 없는 값을 사용하려면 maximum을 더해서 비트 길이를 설저애향 함
  • 예를 들어 32비트 값을 보여주려면
In [79]:
x = -1234
In [80]:
format(2**32, 'b')
Out[80]:
'100000000000000000000000000000000'
In [81]:
format(2**32 + x, 'b')
Out[81]:
'11111111111111111111101100101110'
In [82]:
format(2**32 + x, 'x')
Out[82]:
'fffffb2e'
  • 다른 진법의 숫자를 정수형으로 변환하려면 int() 함수에 적절한 인수 전달
In [83]:
int('4d2', 16)
Out[83]:
1234
In [84]:
int('10011010010', 2)
Out[84]:
1234

토론¶

  • 2진수, 8진수, 16진수 변환은 대개 간단히 해결
  • 숫자와 문자 표현식 사이의 변환을 ㅜ이해 이러한 변환법이 존재한다는 점만 기억
  • 기본적으로는 모두 단일 정수형
  • 8진법을 사용할 때는 프로그래머 주의
In [85]:
import os
In [86]:
!touch 03/test.txt
In [87]:

  File "<ipython-input-87-0ddc7745ffdf>", line 1
    os.chmod('./03/test.txt', 0755)
                                 ^
SyntaxError: invalid token
In [88]:
os.chmod('./03/test.txt', 0o755)
In [89]:
!ls -l 03/test.txt

3.5 바이트에서 큰 숫자를 패킹/언패킹¶

문제¶

  • 바이트 문자열을 unpacking 해서 정수값으로 만들어야 함
  • 매우 큰 정수값을 바이트 문자열로 변환해야 함

해결¶

  • 프로그램에서 128비트 정수값을 담을 수 있는 길이 16바이트 문자열을 다루어야 한다고 가정
In [101]:
data = b'\x00\x124V\x00x\x90\xab\x00\xcd\xef\x01\x00#\x004'
  • 바이트를 정수형으로 변환하려면 int.from_bytes() 사용하고 바이트 순서를 명시
In [102]:
len(data)
Out[102]:
16
In [103]:
int.from_bytes(data, 'little')
Out[103]:
69120565665751139577663547927094891008
In [104]:
int.from_bytes(data, 'big')
Out[104]:
94522842520747284487117727783387188
  • 큰 정수값을 바이트 문자열로 변환하려면 int.to_bytes() 메소드를 사용하고, 바이트 길이와 순서를 명시한다.
In [105]:
x = 94522842520747284487117727783387188
In [106]:
x.to_bytes(16, 'big')
Out[106]:
b'\x00\x124V\x00x\x90\xab\x00\xcd\xef\x01\x00#\x004'
In [107]:
x.to_bytes(16, 'little')
Out[107]:
b'4\x00#\x00\x01\xef\xcd\x00\xab\x90x\x00V4\x12\x00'

토론¶

  • 정수형 값과 바이트 문자열 간의 변환은 일반적인 작업이 아니다. 하지만 네트워크나 암호화가 필요한 특정 애플리케이션에서 사용하는 경우가 존재
  • 예를 들어 IPv6 네트워크 주소는 128비트 정수형으로 표시됨
  • 이 값을 데이터 레코드에서 추출하는 코드를 작성한다면 앞에 나온 기술을 사용해야 함
  • 레시피 6.11에 나오는 struct 모듈을 사용할 수도 있음. 이 방식도 잘 동작하지만, struct로 언패킹할 수 있는 정수형의 크기가 제한적. 따라서 언팩을 여러 번하고 결과 값을 합쳐야 함
In [108]:
data
Out[108]:
b'\x00\x124V\x00x\x90\xab\x00\xcd\xef\x01\x00#\x004'
In [109]:
import struct
In [110]:
hi, lo = struct.unpack('>QQ', data)
In [111]:
(hi << 64) + lo
Out[111]:
94522842520747284487117727783387188
  • 바이트 순서는(little 또는 big)는 정수형을 이루는 바이트가 가장 작은 것부터 표시되었는지 혹은 가장 큰 것부터 표시되었는지를 나타냄
  • 16진수 값을 사용해서 이에 대한 예를 들었다.
In [114]:
x = 0x01020304
In [115]:
x.to_bytes(4, 'big')
Out[115]:
b'\x01\x02\x03\x04'
In [116]:
x.to_bytes(4, 'little')
Out[116]:
b'\x04\x03\x02\x01'
  • 정수형 값을 바이트 문자열로 변환하려는데 지정한 길이에 다 들어가지 않는 경우에는 에러가 발생
  • 이를 방지하기 위해서 int.bit_length() 메소드로 얼마나 많은 비트가 필요한지 미리 검사할 수 있음

3.10 행렬과 선형 대수 계산¶

문제¶

  • 행렬 곱셈, 행렬식 찾기, 선형 방정식 풀기 등 행렬이나 선형 대수 계산을 해야함

해결¶

  • NumPy 라이브러리에 이런 용도로 사용할 수 있는 matrix 객체가 있음
  • 행렬은 레시피 3.9에 나왔던 배열 객체와 비슷한 면이 있지만, 계산할 때는 선형 대수 계산법을 따른다.

import numpy as np
In [259]:

m

        [ 0,  4,  5],
        [ 7,  8, -9]])

# 전치 행렬(transpose)
m.T
Out[261]:
